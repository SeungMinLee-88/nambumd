# 가상화(Virtualization) vs HPC(고성능 컴퓨팅)

## 1. 문서 목적
본 문서는 **공공기관 및 연구기관의 HPC 서비스 도입 검토 및 제안**을 목적으로 작성되었다. 기존 **가상화 기반 IT 인프라**와 **HPC(High Performance Computing) 인프라**를 비교·분석함으로써, 대규모 연산·AI·과학기술 연구에 적합한 컴퓨팅 환경 선택을 지원하고자 한다.

본 문서는 다음과 같은 의사결정을 지원한다.
- 공공·연구기관의 HPC 인프라 도입 타당성 검토
- 기존 가상화 인프라 대비 HPC의 기술적·운영적 차별성 이해
- AI·대규모 계산 연구 과제 수행을 위한 인프라 적합성 평가

---

## 2. 기본 개념

### 2.1 가상화(Virtualization)
가상화는 하나의 물리 서버 자원을 논리적으로 분할하여 다수의 가상 머신(VM) 또는 컨테이너를 실행하는 기술이다. 공공기관의 일반 행정 시스템 및 연구 지원 시스템에서 널리 활용되고 있으며, 자원 활용 효율성과 운영 편의성이 주요 장점이다.

### 2.2 HPC (High Performance Computing)
HPC는 다수의 물리 서버(노드)를 고속 네트워크로 연결하여 대규모 계산을 병렬로 수행하는 컴퓨팅 방식이다. 주로 베어메탈 환경에서 운영되며, 과학·공학 연구, AI 학습, 국가 연구 과제 수행을 위한 핵심 인프라로 활용된다.

---

## 3. 인프라 아키텍처 비교

| 구분 | 가상화 인프라 | HPC 인프라 |
|---|---|---|
| 실행 환경 | 하이퍼바이저 기반 VM | 베어메탈 노드 |
| 자원 할당 방식 | 논리적 분할 공유 | 물리 자원 전용 사용 |
| 네트워크 구성 | 가상 스위치 기반 | 고속 인터커넥트 기반 |
| 작업 처리 방식 | 상시 서비스 운영 | 배치·큐 기반 작업 수행 |
| 확장 구조 | VM 단위 확장 | 노드 단위 병렬 확장 |

---

## 4. 성능 및 기술적 특성 비교

### 4.1 CPU 및 메모리 활용
- **가상화 인프라**는 하이퍼바이저 오버헤드로 인해 연산 성능 저하 가능성이 존재함
- **HPC 인프라**는 CPU 코어 고정(Core Pinning) 및 NUMA 최적화를 통해 최대 성능을 안정적으로 제공함

### 4.2 네트워크 성능
- 가상화 환경은 가상 스위치 계층으로 인해 지연 시간이 증가할 수 있음
- HPC 환경은 초저지연·고대역폭 네트워크를 통해 대규모 병렬 통신에 적합함

### 4.3 스토리지 처리
- 가상화 인프라는 범용 공유 스토리지를 주로 활용함
- HPC 인프라는 병렬 파일 시스템을 통해 대용량 데이터의 고속 입·출력을 지원함

---

## 5. AI 및 연구 계산 관점 비교

| 항목 | 가상화 환경 | HPC 환경 |
|---|---|---|
| GPU 활용 방식 | vGPU 또는 제한적 패스스루 | GPU 직접 연결(Bare Metal) |
| 다중 GPU 확장 | 제한적 | 고속 네트워크 기반 확장 |
| 분산 학습 효율 | 네트워크 병목 발생 가능 | 대규모 분산 학습에 최적 |
| 대형 모델 학습 | 제한적 | 대규모 모델 학습 적합 |
| 연구 프레임워크 | 범용 실행 | MPI/NCCL 최적화 환경 |

---

## 6. 운영 및 서비스 관점 비교

| 항목 | 가상화 기반 서비스 | HPC 기반 서비스 |
|---|---|---|
| 서비스 운영 방식 | 상시 서비스 중심 | 연구 과제 단위 운영 |
| 사용자 대상 | 다수 일반 사용자 | 연구자·전문 인력 |
| 자원 관리 체계 | VM 단위 관리 | 스케줄러·큐 기반 관리 |
| 운영 복잡도 | 비교적 낮음 | 전문 운영 체계 필요 |

---

## 7. 적용 분야 및 활용 사례

### 가상화 인프라 활용 분야
- 행정·업무 정보 시스템
- 연구 지원용 웹 서비스
- 개발 및 시험 환경

### HPC 인프라 활용 분야
- 인공지능·머신러닝 대규모 학습
- 과학·공학 시뮬레이션 연구
- 금융·통계 분석
- 유전체 분석, 기후·환경 모델링

---

## 8. 인프라 선택을 위한 고려 사항

- 가상화 인프라는 범용 정보 시스템 운영에 적합함
- HPC 인프라는 고성능 연산이 요구되는 연구·국가 R&D 과제 수행에 필수적임
- 연구기관의 중·장기 연구 로드맵을 고려한 인프라 선택이 필요함

---

## 9. 결론

HPC 인프라는 공공 및 연구기관의 **첨단 연구 경쟁력 확보를 위한 핵심 기반 시설**이다. 기존 가상화 인프라와의 명확한 역할 구분을 통해, 연구 목적에 부합하는 고성능 컴퓨팅 환경을 구축하는 것이 바람직하다.
