MCP(Model Context Protocol)는 LLM이 외부 도구, 데이터 소스, API 등과 상호작용하는 방식을 표준화하는 프로토콜입니다. 따라서 MCP 없이도 Agentic RAG를 구현할 수 있으며, 이는 특정 프레임워크나 프로토콜에 종속되지 않는 유연성을 제공합니다. 

MCP 없이 Agentic RAG 구현하는 방법

MCP는 Agentic RAG 구현을 위한 여러 선택지 중 하나일 뿐입니다. MCP를 사용하지 않고도, 에이전트의 핵심 기능인 '계획(Planning)'과 '도구 사용(Tooling)'을 구현하면 Agentic RAG 시스템을 구축할 수 있습니다. 

**1. ReAct(Reasoning + Acting) 프레임워크 활용**  
ReAct는 LLM이 추론(Reasoning)을 통해 다음 행동(Acting)을 결정하고, 이 과정을 반복하여 문제를 해결하는 방식입니다. MCP와 같은 표준 프로토콜 없이도 에이전트가 자체적으로 계획을 수립하고 실행하게 합니다. 

- **구현 단계**:
    - **도구 정의**: 웹 검색 API, 코드 실행기, 계산기 등 에이전트가 사용할 수 있는 도구들을 정의합니다.
    - **프롬프트 엔지니어링**: LLM에게 질의에 대한 추론(Reasoning) 과정을 거치고, 적절한 도구(Acting)를 선택하여 사용하도록 지시하는 프롬프트를 작성합니다.
    - **반복 실행**: LLM의 응답에 따라 도구를 호출하고, 그 결과를 다시 LLM에 입력하여 다음 행동을 결정하는 과정을 반복합니다. 

**2. LangChain 또는 LlamaIndex 활용**  
LangChain과 LlamaIndex는 Agentic RAG를 구축하기 위한 오픈소스 프레임워크입니다. MCP 없이도 내장된 도구와 에이전트 기능을 활용해 복잡한 워크플로우를 구성할 수 있습니다. 

- **LangChain Agents**: 사용자가 정의한 도구들을 에이전트에 연결하고, LLM이 어떤 도구를 언제 사용할지 판단하게 합니다.
- **LlamaIndex Agents**: LlamaIndex의 에이전트는 다양한 데이터 소스와 연동하여 검색 및 분석 작업을 수행하도록 설계되었습니다. 

**3. 수동 도구 연동**  
MCP가 도구와 LLM 간의 통신을 표준화하는 반면, MCP 없이도 각 도구의 API를 LLM에 직접 연동하여 에이전트 기능을 구현할 수 있습니다. 

- **구현 단계**:
    - **API 호출 함수 작성**: 필요한 외부 서비스(예: 구글 검색, 사내 DB)의 API를 호출하는 파이썬 함수를 만듭니다.
    - **LLM에 도구 제공**: LLM에 도구의 기능과 사용법을 설명하는 프롬프트(Tooling)를 제공하고, LLM이 응답 시 해당 함수를 실행하도록 합니다.
    - **결과 통합**: 함수 실행 결과를 다시 LLM에 입력하여 최종 답변을 생성하게 합니다. 

MCP와 Agentic RAG의 관계

MCP는 Agentic RAG의 필수 요소가 아니라, Agentic RAG 시스템을 구축할 때 유용하게 사용할 수 있는 표준 프로토콜입니다. 

|비교 항목|MCP 기반 Agentic RAG|MCP 없는 Agentic RAG|
|---|---|---|
|**도구 연동 방식**|MCP 서버를 통해 표준화된 방식으로 외부 도구와 통신.|LangChain, LlamaIndex 등을 통해 도구를 직접 연동하거나, 수동으로 API를 연결.|
|**시스템 유연성**|MCP 표준을 따르는 다양한 시스템과 쉽게 결합 가능.|특정 프레임워크나 직접 구현한 코드에 종속되므로 재사용성이 낮을 수 있음.|
|**개발 복잡성**|초기 MCP 서버 구축이 필요하지만, 이후 도구 연동이 단순해짐.|MCP 서버가 필요 없지만, 각 도구별 API 연동 로직을 개별적으로 구현해야 함.|

결론

MCP 없이 Agentic RAG를 구현하려면 ReAct 프레임워크를 기반으로 LLM이 도구 사용을 계획하고 실행하는 로직을 직접 구현하거나, LangChain이나 LlamaIndex 같은 프레임워크의 에이전트 기능을 활용하면 됩니다. MCP는 에이전트와 도구 간의 통신을 표준화하는 역할일 뿐이므로, 다른 방법으로도 Agentic RAG의 핵심 기능을 충분히 구축할 수 있습니다.
